My Second Year
Projects

During my second year at Staffordshire University I aim to create many frameworks and games
which are displayed here.

Multiplayer Games Programming

Pong:

During Multiplayer Games Programming we are tasked with creating a video game that makes use of multiple clients, different packet types, and a server to deal with this information. The program should successfully synchronise both clients to display the same information and give off the illusion that they are occurring simultaneously.


Real Time Rendering

Creating a 3D environment using DX11:

During the first semester of Year 2, we have been tasked with using Direct X 11 to create a 3D environment that is capable of displaying different objects using lighting and texturing to enhance the experience. Several different shapes are rendered and orbit one another whilst also rotating, behaving similarly to our solar system. There are different lighting and texture loading techniques present in this model.


Applied Mathematics

Creating a physics-based environment using mathematics:

The aim of this module was to develop an environment that simulates movement of physical objects whilst taking into account different forces and physics systems. For instance, cubes can slide across the floor but will feel the resistant force of friction, or can fly into the air only to be brought back down via gravity. Each object also makes use of AABB (Axis aligned bounding box) colliders which allow them to bounce off one another and push eachother in any given direction.


Game Artificial Intelligence

Creating several AI systems such as pathfinding and decision making:

For this module, my aim was to implement multiple different artifial intelligence techniques to create appropriate enemy behaviour. This includes behaviours such as seeking to a point, evading a player and chasing a moving target, or a combination of several techniques. I also experimented with different path finding techniques such as Djikstra's Algorithm and A* Search, making use of different methods like Manhattan. These were then evaluated to see which one calculates the best possible route to a location whilst also using the least amount of computational resources.


Tools Development

Creating a procedural terrain generator through the Perlin Noise function:

Through the Unity Engine and making use of the Perlin Noise function, terrain can be procedurally generated through a custom-made tool. Terrain can be generated at variable sizes and amplitutes as either an array of cube objects, similar to Minecraft, or by using meshes taht draw a smoother environment. The user has several different parameters they can play around with, such as the grid size, the maximum height of the map, and whether the terrain is flatter or more rough.


Junior Collaborative Development

Collaborating with a group of people to create a game from scratch on Unreal:

As part of a larger team, my responsibility as a Junior Programmer was to implement some of the more basic yet essential systems used in our collaborative project. On the Unreal engine I created several different particle effects such as a blood splatter and sword slash effect. Additionally, I was tasked with creating a sufficient movement and stamina system that punishes the player for attacking or running too much, with a visual stamina bar presented to the screen.
